### Device Health Check

> **Important**: `npu-smi info` showing `Health=OK` is **unreliable** — hung NPUs still report OK.
> The only reliable detection is an actual tensor operation with synchronization.
> Timeout must be >= 30s (torch + torch_npu cold start takes ~18s).

**Step 1 — Get device count** (on host):
```bash
ssh npu_server "npu-smi info | grep -cE '^\| [0-9]'"
```

**Step 2 — Parallel test all devices** (in container, 30s timeout each):
```bash
COUNT=<from step 1>
for N in $(seq 0 $((COUNT - 1))); do
  (
    out=$(ssh npu_server 'docker exec cjm_cann1 bash -c "cd /home/c00959374/AscendOpGenAgent && timeout 30 python3 -c \"
import torch, torch_npu
torch_npu.npu.set_device('$N')
(torch.ones(1).to('"'"'npu:'$N''"'"') + 1)
torch_npu.npu.synchronize('$N')
print('"'"'OK'"'"')
\""' 2>/dev/null)
    [ "$out" = "OK" ] && echo "npu:$N OK" || echo "npu:$N HUNG"
  ) &
done
wait
```

**Step 3 — Report results**: list status per device, suggest first available. If all HUNG, report "all NPUs unavailable, contact admin" and do not proceed.

## Remote File Transfer

**Method 1: SCP (recommended)**
```bash
# Local file → remote host
scp local_file.sh npu_server:/remote/path/

# Local file → remote Docker container
scp local_file.sh npu_server:/tmp/ && \
  ssh npu_server 'docker cp /tmp/local_file.sh cjm_cann1:/remote/path/'
```

**Method 2: HERE document (prevents local variable expansion)**
```bash
ssh npu_server 'cat > /path/to/file.sh << "EOF"
#!/bin/bash
# Quoted "EOF" prevents local shell from expanding $VAR
echo "Hello"
EOF'
```

**Method 3: Base64 encoding (fallback for complex content)**
```bash
# Step 1: Encode locally
base64 input.sh > /tmp/encoded.b64
# Step 2: Transfer
scp /tmp/encoded.b64 npu_server:/tmp/
# Step 3: Decode on remote
ssh npu_server 'base64 -d /tmp/encoded.b64 > /final/path/output.sh'
```
**Avoid**: `echo "b64..." | base64 -d > file` — shell special character handling is unreliable.
